## üöÄ How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. Check if the issue already exists in the [Issues](https://github.com/hs14235/hs14235.github.io/issues) page
2. If not, create a new issue with:
   - Clear, descriptive title
   - Detailed description of the problem or suggestion
   - Steps to reproduce (for bugs)
   - Screenshots if applicable
   - Your environment (browser, OS, etc.)

### Making Changes

1. **Fork the Repository**
   ```bash
   git clone https://github.com/hs14235/hs14235.github.io.git
   cd hs14235.github.io
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-bug-fix
   ```

3. **Make Your Changes**
   - Follow the existing code style
   - Test your changes locally
   - Ensure HTML validates
   - Check accessibility compliance
   - Test on multiple browsers if possible

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "feat: Add new feature" # or "fix: Fix bug"
   ```

5. **Push and Create Pull Request**
   ```bash
   git push origin feature/your-feature-name
   ```
   Then create a Pull Request on GitHub.

## üìù Commit Message Guidelines

Follow conventional commits format:

- `feat:` - New feature
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style changes (formatting, etc.)
- `refactor:` - Code refactoring
- `perf:` - Performance improvements
- `test:` - Adding or updating tests
- `chore:` - Maintenance tasks

Example:
```
feat: Add dark mode toggle
fix: Correct contact form validation
docs: Update README with installation steps
```

## üé® Code Style Guidelines

### HTML
- Use semantic HTML5 elements
- Include proper ARIA labels for accessibility
- Maintain consistent indentation (2 spaces)
- Use descriptive IDs and classes
- Keep HTML clean and readable

### CSS
- Follow existing naming conventions
- Use meaningful class names
- Group related styles together
- Comment complex CSS when necessary
- Maintain responsive design principles

### JavaScript
- Use ES5+ syntax for consistency with existing code
- Add comments for complex logic
- Follow existing jQuery patterns
- Ensure cross-browser compatibility
- Test all interactive features
